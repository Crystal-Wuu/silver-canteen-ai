# 数据库复刻与使用注意事项

本项目实际使用的数据库由以下三个 SQL 文件建立。请严格按照顺序执行，不需要执行其他数据脚本。

## 一、执行前准备

1. 安装 SQL Server 和 SQL Server Management Studio（SSMS）。
2. 打开 SSMS，并连接到准备运行本项目的 SQL Server 实例。
3. 新建数据库，数据库名称必须为 `Subsidy_system`：

```sql
CREATE DATABASE Subsidy_system;
GO
```

如果已经存在同名数据库，并且其中有需要保留的数据，请先备份。后续的完整数据恢复脚本会清除八张业务表中的已有数据。

## 二、按顺序执行三个脚本

### 第一步：执行 `sql/建表.sql`

此脚本会：

- 删除已有业务表；
- 重新创建老人、助餐点、补贴资格、补贴规则、就餐记录、结算单、异常预警和审计日志表；
- 创建视图、存储过程等数据库对象；
- 插入少量用于验证表结构的初始数据。

执行成功后，SSMS 消息窗口不应出现红色错误。

注意：该脚本会删除并重建业务表，因此不要在需要保留数据的正式数据库中执行。

### 第二步：执行 `sql/触发器代码.sql`

此脚本会创建或更新以下业务触发器：

- 就餐记录插入时自动计算补贴并识别异常；
- 老人账户状态与补贴资格联动；
- 补贴资格和补贴规则审计；
- 结算单审核状态联动。

脚本使用 `CREATE OR ALTER TRIGGER`，可以重复执行。

### 第三步：执行 `sql/完整数据导出.sql`

这是最终数据恢复脚本，也是复刻数据库最关键的一步。脚本会自动：

1. 开启事务；
2. 临时停用业务触发器；
3. 按外键依赖顺序删除建表脚本产生的初始数据和此前已有数据；
4. 导入最终完整数据；
5. 重新启用触发器；
6. 提交事务；
7. 输出数据量和页面核心指标核验结果。

临时停用触发器非常重要。完整数据属于已经计算好的历史快照，如果导入时触发器保持工作，就餐记录中的补贴金额和结算状态可能被重新计算，并可能额外生成异常预警，导致数据量与项目原始结果不一致。

脚本使用事务执行。如果中途任意语句报错，所有删除和导入操作会自动回滚，不会留下只导入一半的数据。

## 三、正确的数据量

完整数据导入后，脚本底部会自动输出核验结果：

| 数据表 | 正确记录数 |
| --- | ---: |
| Elderly | 50 |
| DiningPoint | 5 |
| SubsidyRule | 4 |
| SubsidyQualification | 50 |
| DiningRecord | 2363 |
| SettlementSheet | 45 |
| AbnormalWarning | 24 |

页面核心指标应为：

| 指标 | 正确结果 |
| --- | ---: |
| 正常账户老人 | 45 |
| 累计就餐记录 | 2363 |
| 累计补贴金额 | 18136.60 |
| 待处理预警 | 14 |
| 正常运营助餐点 | 5 |

其中“待处理预警”统计的是 `HandleStatus` 为“待处理”或“处理中”的记录，并不是异常预警表的全部 24 条记录。

## 四、常见问题

### 提示数据库不存在

说明尚未创建 `Subsidy_system`，或者脚本顶部的数据库名称与实际名称不一致。先创建同名数据库再执行脚本。

### 提示对象或表不存在

通常是没有先执行 `建表.sql`，或者某一步执行失败。重新按照“建表、触发器、完整数据导出”的顺序执行。

### 提示主键重复

说明使用的是旧版完整数据脚本，或者没有清空原数据。当前版本的 `完整数据导出.sql` 会在导入前自动清空，可直接重新执行。

### 导入后预警数量过多或补贴金额不同

通常是导入历史数据时触发器仍在运行。请使用当前版本的 `完整数据导出.sql`，它会自动临时停用并恢复触发器。

### 中文显示乱码

请使用 SSMS 打开文件，并确保脚本以 UTF-8 编码保存。数据脚本中的中文字符串使用 `N'中文内容'` 格式，不能删除前面的 `N`。

### 网站显示数据库未连接

检查项目根目录 `.env` 中的配置，特别是：

```ini
DB_SERVER=你的SQL Server服务器名称
DB_NAME=Subsidy_system
DB_TRUSTED_CONNECTION=yes
```

然后访问 `http://127.0.0.1:5000/api/test_db` 查看具体错误。

## 五、日常使用注意事项

- 不要在导入完整数据后再次单独执行 `建表.sql`，否则所有表和数据会被重新创建。
- 需要重新恢复初始状态时，依次重新执行三个脚本即可。
- `完整数据导出.sql` 会覆盖现有业务数据，执行前必须确认数据库中没有需要保留的新数据。
- 完整数据导入结束后触发器会恢复工作，之后新增就餐记录时会正常自动计算补贴和生成预警。
- 网站使用的数据库名称必须与脚本中的 `Subsidy_system` 完全一致。
